# Terraform Samples

These samples show a cloud foundation shape for AI modernization. They are intentionally provider-light and use placeholders that must be integrated into an enterprise landing zone before production use.

## Included

- Object storage for lakehouse zones
- Managed Kafka-compatible cluster placeholder
- Kubernetes namespace integration variables
- Tags and naming conventions

## Production Hardening

- Add remote state with locking.
- Enforce policy-as-code checks.
- Use private endpoints and approved network modules.
- Bind IAM to enterprise identity groups.
- Add secret manager integration.
- Enable audit logs and retention policies.
